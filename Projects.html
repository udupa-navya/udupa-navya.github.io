Personal Project: Shared Grocery List App (ongoing)

Technologies: Java11, SpringBoot (Backend), JavaScript, HTML5 (Frontend), Firebase (Database)

A fully functioning shared grocery list web application prototype. It includes:

  1. Real-time list sharing
  2. User authentication
  3. Notification system

Product Documents:

  1. PRP (Product Requirements Plan)
  2. MRD (Market Requirements Document)
  3. PRD (Product Requirements Document)

Documentation: A detailed report and PowerPoint presentation covering:

  1. Introduction, Problem Statement, Objectives, Deliverables, etc.
  2. Market Research, Target Market, Competitor Analysis
  3. User Personas, Core Features, UX, and Wireframes
  4. Business Model Canvas, Technical Requirements, and Architecture

Amalyzed hypothetical user data using Power BI to derive actionable insights for product strategy. Below are the insights:

  • Count of sign-up dates by membership status: Highlights user interest in different membership tiers.
  • Sum of age by living conditions: Identifies dominant age groups in shared spaces.
  • Sum of household size by occupation: Correlates user demographics with app functionality.

These insights informed the app's focus on collaboration, duplicate detection, and list-sharing features.
